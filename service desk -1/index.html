Weekly Update

(17 - 21) Feb 2020

  1. If we receive any LYNC ticket for assigning a new phone number, steps that we need to follow :- • Check the Account in LYNC server , if its enabled or not • If the account is enabled and having a LINE URI having a phone number, assign the ticket to Collaboration team to work on it
  2. For M-files access or CLM license: • Add the user account to “LG_M_Files_SanomaCLM” • After adding the group assign the RITM to Accenture Data FI team who will provide access to Sanoma CLM
  3. Please ensure to delete/remove the alternate VPN client group whenever we receive any request to add user to any specific VPN Client group since users cannot have 2 VPN groups at the same time. Example of specific VPN group: • Client_VPN_users_Sanoma_Basic_Allow_Local_LAN • Client_VPN_users_ext_Mystes
  4. Sanoma Pro SAP Access: If we receive a request for Sanoma Pro SAP access ensure to follow the below process. • This request needs to be raised in Jira by key user present at user location. • If user is aware of the system where access is required same can be mentioned on Jira ticket. • If user is un-sure of the system where access is required they can always mention a reference user who already has access. • Inform the above points to user and resolve the request at your end. KB Number :- KB0012307 KB Name :- 1 - Global - Sanoma Pro SAP access
  5. SAP GUI package must be installed if there is a request for SAP Analysis for Office or any other SAP application
  6. If we receive a Prolong account request for an employee with a personnel number (Import data scource = SAP HCM/HR), we ask user to contact HR
